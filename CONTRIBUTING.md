## 一、贡献语料（经文文件）

### 放置位置

根据优先级放入对应子目录：

```
corpus/
├── 第一级/    ← 金刚经、维摩诘经、圆觉经、楞严经、法华经
├── 第二级/    ← 解深密经、涅槃经、胜鬘经、无量寿经、药师经
├── 第三级/    ← 地藏经、心经、华严入法界品等
└── 梵巴原典/  ← Sanskrit / Pāli 文本
```

### 文件命名规范

```
{经名}_{译者}_{版本}.{扩展名}

示例：
金刚般若波罗蜜经_鸠摩罗什_CBETA-T08n0235.xml
维摩诘所说经_鸠摩罗什_CBETA-T14n0475.xml
```

### 格式要求

**优先**：XML TEI 格式（来自 CBETA，保留完整标注）

**也接受**：纯文本（`.txt`），要求：
- UTF-8 编码
- 保留品名标题（如「◎卷上」「第一品」）
- 不要混入现代标点以外的额外注释

**不接受**：
- 扫描图片、非结构化 PDF
- 已有显著编辑修改的版本
- 来源不明的文本

### 每个文件须附 `_source.md` 说明

在同目录新建一个同名 `_source.md` 文件，写明：

```markdown
# 来源说明

- **经名**：金刚般若波罗蜜经
- **译者**：鸠摩罗什（姚秦）
- **大藏经编号**：T08n0235
- **来源**：CBETA 中华电子佛典协会
- **版本日期**：CBETA 2024
- **原始 URL**：https://cbetaonline.dila.edu.tw/zh/T0235
- **版权说明**：依 CBETA 学术非商业使用条款使用，需注明来源
```

## 二、贡献引文（核心引文库）

`佛陀/references/核心引文库.md` 是 skill 的「内建记忆」。向此文件添加精华段落，可以直接增强佛陀的引用准确度和主题覆盖。

### 选段标准

✅ **适合收录**：
- 佛陀直接说话的段落（非菩萨、弟子的独白）
- 与现代人常见困惑直接相关（焦虑、执念、死亡、人际、意义感）
- 100字以内的精华段落（太长的段落请截取核心句）
- 有明确经典出处可标注

❌ **不适合收录**：
- 仪轨、陀罗尼、神咒类文字
- 高度技术性的论辩（如唯识细分义）
- 来源不明的「伪经」或民间流传版本

### 格式规范

在对应主题下新增，格式如下：

```markdown
**《经名》**（译者，可选）
> 「原典语句。」
> 「若有多句，每句一行。」
```

如需新增主题，在文件末尾添加 `## 七、新主题名` 并说明适用场景。

## 三、贡献梵文/巴利文原典

本项目预留梵巴扩展接口，目前待填充：

- `corpus/梵巴原典/`：存放原文文件
- `佛陀/references/梵巴引文库.md`：存放对照引文

### 格式要求

梵文引文请同时提供：
1. IAST 转写（标准学术转写）
2. 对应汉译经名及段落

巴利文引文请同时提供：
1. 巴利原文（VRI 版或 PTS 版，注明来源）
2. 对应汉译经名（阿含/尼柯耶对应）

**参考格式**（`梵巴引文库.md` 中）：
```markdown
## 「应无所住，而生其心」对应原典

**汉文**（《金刚经》，鸠摩罗什译）：
> 「应无所住，而生其心。」

**梵文**（IAST）：
> api tu khalu punaḥ Subhūte bodhisattvena mahāsattvena na kvacit pratisthitena dānaṃ dātavyam

**来源**：Vajracchedikā Prajñāpāramitā, ed. Conze (1957), §10
```

## 四、反馈角色问题

如果你觉得佛陀的某个回答：
- 语气太像 AI / 心理咨询师
- 引用原典有误
- 使用了不该使用的现代词汇
- 某类问题始终回答得不好

请提交 **Issue**，选择模板「角色反馈」，并提供：
1. 你的原始提问
2. 佛陀的实际回答（截图或文本）
3. 你认为问题所在
4. 如果有，你认为更合适的回应方向

## 五、改善 SKILL.md 或对话范式

`SKILL.md` 是整个 skill 的核心。修改前请先提 Issue 说明思路，与维护者确认方向后再提 Pull Request，避免方向性冲突。

对话范式（`references/对话范式.md`）欢迎直接贡献新的示例对话，格式参照现有范式即可。

## 六、提交流程

1. **Fork** 本仓库
2. 新建分支：`git checkout -b contribute/经名或功能描述`
3. 做出修改，确保符合本文档的格式规范
4. 提交：`git commit -m "feat: 添加《解深密经》语料"` 或 `"fix: 修正引文库中涅槃经引文"`
5. 发起 **Pull Request**，简要说明你做了什么、来源是什么

### 提交信息前缀规范

| 前缀 | 用途 |
|------|------|
| `feat:` | 新增语料、引文、功能 |
| `fix:` | 修正错误（引文有误、格式错误）|
| `refactor:` | 改善 SKILL.md 或范式逻辑 |
| `docs:` | 更新文档 |

## 七、版权与许可须知

**在提交任何文本语料之前，请确认：**

- [ ] 该文本来源明确（CBETA、SAT、PTS 等学术数据库）
- [ ] 符合来源数据库的使用条款（一般为非商业学术使用）
- [ ] 已在 `_source.md` 中注明所有来源信息
- [ ] 未包含任何现代学者的翻译或注释（现代译文受版权保护）

**本项目不接受**：
- 来源不明的文本
- 需要付费授权的商业数据库内容
- 任何声称是「最新发现」或「真实密传」的非公认经典

如有版权疑问，提 Issue 询问后再提交。
